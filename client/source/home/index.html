What is Liwe?

Liwe is a service for web developers to use smartphones as a remote for their webapp. No need of an extra servers or plugins, just JavaScript. Giving you the freedom to use new methods of interaction, with your web apps and users, limited only by your imagination.

Even my mom’ can use it

One of the main objectives of Liwe, was to be easy to use and interoperable. So what better to achieve this than HTML and Javascript? The idea is to make it as simple as the end user to visit a webpage and nothing else. No apps to download, no registration, no princess to save.

Let’s make it quick! Users don't like to wait.

Noobs welcomed

On the devs side, the project aims to be as simple as possible. Anybody should be able to use it. No need of an extra server, no need to learn a new language. Just insert our script in your webapp, and you’re ready to use our API.

Best of all, it's JavaScript powered!

I WANT YOU

Liwe is open source, and built on open source technologies. Unfortunately, the project is currently closer to a prototype than production. This is for a few reasons: no proper protocol defined, use old node packages, lack of flexibility and security, and not a single test. This is why I want you devs to rethink the project to (re)build it correctly.
Join the conversation on Gitter and tell me what you think.

Your feedback is really important, even if you're not a developer. Users are more than welcome to tell us about their experience, about what they like or don't, and which improvement they would like to see. As long as the critic is constructive, it will be welcomed. Of course we don't expect them to create a GitHub account to share their point of view, but a blog post, a mail or a tweet is enough.
